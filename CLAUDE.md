# TaskQuest

モバイルアプリプロジェクト。タスク/クエスト管理系のアプリを開発予定。

## プロジェクト状況

- アプリ概要: ギルド世界観のゲーミファイドタスク管理アプリ
- フレームワーク: React Native CLI (v0.85.3) + TypeScript
- 対象プラットフォーム: iOS / Android（まず Android APK から）
- 初期方針: オフライン完結（サーバなし）

## 開発方針

- [x] アプリコンセプト確定
- [x] 技術スタック選定
- [x] 初期セットアップ

## 技術スタック

- React Navigation (bottom-tabs + stack)
- Zustand + AsyncStorage (状態管理 + ローカル永続化)

## ディレクトリ構造

```
src/
  types/         # TypeScript 型定義
  constants/     # テーマ、ゲームデータ
  store/         # Zustand ストア
  components/    # 共通 UI コンポーネント
  screens/       # 各画面
  navigation/    # ナビゲーション設定
```

## コーディング規約

- 画面コンポーネントは `src/screens/` に配置
- 共通 UI は `src/components/common/` に配置
- スタイルは `StyleSheet.create` でファイル末尾にまとめる
- テーマ色は `Colors` から参照、直書き禁止

## コンセプト

一般的なRPGではシステムが用意されているが、ユーザーが自分でクエストを発注できる点が特徴。タスクの達成度に応じてレベルアップや報酬が得られる。
RPGのギルドの世界観を重視する。

2Dのドット絵で、シンプルなUIを目指す。

**対象ユーザー:** 個人タスク管理用途（将来的にソーシャル/マルチプレイヤー機能に拡張予定）

## 主な機能

### コア機能（フェーズ1）
- クエスト(タスク)の発注
- クエスト(タスク)の受注
- クエスト(タスク)の達成度管理
- クエスト(タスク)の報告システム
- レベルアップシステム
- 報酬システム
- アイテム、装備の売買
- ガチャシステム

### レベルアップのメリット
- ギルドランク制 — ランクアップで受けられるクエストの難易度・報酬枠が増える
- ギルドホームのアンロック — レベルに応じてホーム画面の背景（ドット絵）が変化
- 称号・肩書き — 「見習い冒険者」→「銀の剣士」などプロフィールに表示
- ガチャ回数の解放 — レベルアップで1日のガチャ回数枠が増える

### バトルシステム（ゲームモード）
タスクをこなして強化したキャラで闘技場に挑む。タスク継続のモチベーションの核となる機能。
- 日常タスク → 経験値・装備入手 → キャラ強化 → 闘技場で腕試し
- フェーズ1: CPU戦（オフライン）
- フェーズ2以降: オンライン対戦（拡張時）

## 開発の優先事項

- スマホで完結させることを重視
- まず個人利用として完成させ、その後ソーシャル機能を拡張する方針

## 追加機能(済)
- 装備の合成機能 — 同じ装備を4つ合成してレアリティアップ
- 闘技場モード追加
  - オートバトル（ターン制自動進行、スキップ機能付き）
  - アリーナランク制（新人→銅→銀→金→白金→伝説）勝利数で昇格
  - アリーナショップ（専用コインで限定装備・称号を購入）
  - 各ランク帯に3体の対戦相手（難易度別）

## 修正機能(済)
- ガチャ回数は無制限に変更
- ランクに応じてガチャのレアリティ枠、確率が増える仕様に変更
- クエスト発注の項目は「クエスト名」「難易度」「クエスト内容」「達成条件」「報酬」「期限」の4つに変更
